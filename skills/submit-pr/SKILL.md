# PR提出

## 概要

PRの本文をプロジェクトのテンプレートに従って作成・更新する。

## 使用するコマンド

```bash
# PR本文の更新
gh pr edit <PR番号> --body "$(cat <<'EOF'
<本文>
EOF
)"

# 既存PRの確認
gh pr view <PR番号>
```

## テンプレート

```markdown
## 関連 Issue

close #<Issue番号>

## 概要

<実装した機能や修正した内容を具体的に記述>

### 変更内容

- `<ファイルパス>`: <変更内容>
- `<ファイルパス>`: <変更内容>

## 動作確認方法

1. <確認手順1>
2. <確認手順2>
3. <確認手順3>

## 伝達事項

- <他の開発者への重要な情報や注意点>
```

## 関連 Issue の書き方

| 種類 | 書き方 | 効果 |
| --- | --- | --- |
| 自動クローズ | `close #123`, `fix #123`, `resolve #123` | マージ時にIssueを自動クローズ |
| 参照のみ | `#123`, `ref #123` | Issueへのリンクのみ |

## チェックリスト

PR本文作成時に確認:

- [ ] 関連Issueが記載されている
- [ ] 概要が具体的に記述されている
- [ ] 変更ファイルが一覧化されている
- [ ] 動作確認方法が記載されている
- [ ] 必要に応じて伝達事項が記載されている
